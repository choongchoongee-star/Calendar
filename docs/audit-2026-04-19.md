## 3. 명확한 오류 → 이미 수정 완료

### 3.1 `supabaseAuthToken` dead code 제거 (완료)
Phase 4 Firebase 마이그레이션 이후에도 위젯 브릿지에 `authToken` 파라미터가 JS→네이티브로
전달되고 있었고, 위젯은 `supabaseAuthToken` 키로 UserDefaults에 저장하고 있었음.
위젯 측 `getAuthToken()`은 정의만 돼 있고 **어디서도 호출되지 않음** (grep 전체 검색 0 건).

- **수정:** `docs/script.js:99` — `authToken` 필드 제거
- **수정:** `ios/App/App/WidgetBridge.swift` — `authToken` 읽기와 `supabaseAuthToken` 쓰기 제거
- **수정:** `ios/App/App/WidgetSource/CalendarWidget.swift` — `authTokenKey`, `getAuthToken()` 제거

### 3.2 SPEC §2 줄 수 현실화 (완료)
`script.js`: ~1,640 → ~1,690줄, `CalendarWidget.swift`: ~640 → ~680줄.

## 4. 사용자 확인 결과 반영 (2026-04-19 추가 커밋)

### 4.1 SPEC §2 폴더 구조 축약 → ✅ 해결
**결론:** "주요 파일만 표시" 주석을 SPEC §2 헤더에 추가. AppDelegate.swift / Info.plist / App.entitlements / PrivacyInfo.xcprivacy / WidgetBridge.m / config.xml / exportOptions.plist 등은 의도적 생략으로 확정.

### 4.2 `vibe://` URL scheme → ✅ 해결 (`chaeuda://`로 리네임)
**결론:** `chaeuda://`로 리네임. 변경된 파일:
- `ios/App/App/Info.plist` `CFBundleURLSchemes`
- `ios/App/App/WidgetSource/CalendarWidget.swift` 3 곳
- `docs/script.js` `window.handleDeepLink` (`vibe:` 폴백 유지 — 기존 위젯 호환)
- `SPEC.md` / `README.md` 표기 업데이트

### 4.3 Google OAuth Client ID 하드코딩 → ⏳ 사용자 확인 대기
> **🔔 리마인더:** 사용자가 Firebase 콘솔에서 iOS OAuth Client ID
> `957602147466-rr3skbl5besmar1u6minitknhm1bpriv.apps.googleusercontent.com`가
> 번들 ID `com.dangmoo.calendar`로 **제한되어 있는지** 확인 필요.
>
> - 번들 ID 제한이 켜져 있으면 → 현행 유지(A)
> - 제한 없으면 → CI 주입으로 이전(B). Info.plist 내 `GIDClientID` 값과
>   `com.googleusercontent.apps.*` reverse scheme 모두 치환 대상
>
> 확인 방법: Google Cloud Console → APIs & Services → Credentials → iOS client 선택 → Bundle ID 필드에 `com.dangmoo.calendar` 있는지 확인.

### 4.4 Apple 팀 ID 노출 → ✅ 해결
**결론:** 현행 유지. Apple 공식 관행이며 공개값으로 분류됨.

### 4.5 SPEC §4.1 계정 탈퇴 공유 달력 처리 → ✅ 해결 (경고 다이얼로그 추가)
**결론:** 탈퇴 버튼 onclick에 pre-flight 체크 추가. 사용자가 소유한 달력 중 타 멤버가 있는 경우,
"⚠️ 공유 달력 N개가 삭제되며, 초대된 M명이 접근 불가해집니다" 경고 후 informed consent로 진행.

### 4.6 2031+ 공휴일 → ✅ 해결 (CI 연 1회 자동 갱신)
**결론:** `.github/workflows/holiday-fetch.yml` + `scripts/fetch-holidays.js` 도입.
행안부 특일정보 API(data.go.kr, 무료)로 연 1회 5년치 갱신 → PR 자동 생성.
GitHub Secret `DATA_GO_KR_API_KEY` 등록 필요.
